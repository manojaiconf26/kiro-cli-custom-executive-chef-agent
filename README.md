# South Indian Executive Chef System

An AI-powered conversational multi-agent system built on the Kiro platform that suggests authentic South Indian recipes through natural chat interaction. The Executive Chef Agent generates recipes from its culinary knowledge, enforces regional authenticity, manages a recipe cache, and delegates preparation instructions to a Sous Chef Agent.

## How It Works

1. User describes what they want in Kiro chat — meal type, dietary needs, or available ingredients
2. Executive Chef checks the `recipes/` cache for matches, generates new options from culinary knowledge
3. Presents a top-5 numbered list mixing cached and new recipes
4. User selects a recipe → agent generates the full recipe and saves to `recipes/`
5. A post-tool hook automatically triggers the Sous Chef Agent to generate `prep-list.md`
6. If an ingredient is unavailable, the Substitution Skill finds authentic South Indian alternatives

## Agents

### Executive Chef Agent

The primary conversational agent. Configured in `.kiro/agents/executive-chef.json`.

- Suggests 5 authentic South Indian recipes per request
- Covers Tamil Nadu, Kerala, Karnataka, Andhra Pradesh, and Telangana
- Enforces a blocked ingredient list — no generic blends (curry powder, garam masala, mixed spice powder, all-purpose seasoning)
- Checks `recipes/` cache before generating new recipes
- Invokes the Substitution Skill when ingredients are unavailable
- Logs every interaction to `logs/transaction-log.md`
- Has a `postToolUse` hook on `fs_write` that triggers Sous Chef orchestration

### Sous Chef Agent

Preparation and mise en place specialist. Configured in `.kiro/agents/sous-chef.json`.

- Triggered automatically after a recipe file is written
- Reads the recipe, generates `prep-list.md` with exact measurements and timed steps
- Converts vague amounts to measurable quantities
- Logs each prep-list generation to `logs/transaction-log.md`

## Substitution Skill

Defined in `.kiro/steering/skills/substitution-skill.md`. Invoked on-demand when a user says an ingredient is unavailable.

Supported substitution mappings:

| Missing Ingredient | Alternatives |
|---|---|
| Tamarind | Kokum (0.5x), Amchur (0.3x) |
| Coconut milk | Cashew paste (0.75x), Poppy seed paste (0.5x) |
| Curry leaves | Bay leaves (0.2x) |
| Mustard seeds | Cumin seeds (1:1) |
| Urad dal | Chana dal (1:1) |
| Jaggery | Palm sugar (1:1) |
| Coconut oil | Sesame oil / gingelly (1:1) |
| Toor dal | Moong dal (1:1) |
| Asafoetida (hing) | Onion powder (4x) |
| Fresh coconut | Desiccated coconut (0.5x) |

Unmapped ingredients return an empty list — the agent tells the user no authentic substitute is available.

## Recipe Template

Every recipe file in `recipes/` follows this mandatory 6-section structure:

```markdown
# {Recipe Name}
## Regional Classification
## Dietary Tags
## Ingredients
## Substitutions Applied
## Steps
```

The `parseRecipeTemplate()` function in `src/utils/recipe-template.ts` validates completeness. Missing any section makes the recipe invalid.

## Utility Functions

### `src/utils/validation.ts`
- `validateIngredients(ingredients)` — checks an ingredient list against the blocked set (case-insensitive)
- `getBlockedIngredients()` — returns the blocked ingredient set

### `src/utils/recipe-template.ts`
- `parseRecipeTemplate(markdown)` — parses and validates a recipe markdown file
- `formatRecipeTemplate(recipe)` — converts a RecipeTemplate object back to markdown

### `src/utils/substitution.ts`
- `lookupSubstitution(ingredientName)` — returns all alternatives with conversion ratios
- `applySubstitution(original, alternative)` — replaces ingredient and adjusts quantity
- `getMappedIngredients()` — lists all ingredients that have substitution mappings

## Transaction Logging

All agent interactions are logged to `logs/transaction-log.md` with structured entries including:
- Timestamp and agent name (Executive Chef or Sous Chef)
- Request summary, files read/written
- Recipes suggested and selected
- Substitutions applied
- Errors encountered

## Kiro Configuration (`.kiro/`)

### Agent Definitions

#### `executive-chef.json`
- Tools: all (`*`), with `read` and `write` allowed
- Resources: `src/utils/validation.ts`, `src/utils/recipe-template.ts`, `src/utils/substitution.ts`, `recipes/**/*.md`, `logs/transaction-log.md`, and the substitution skill
- Hook: `postToolUse` on `fs_write` — triggers mandatory Sous Chef invocation after every recipe file write
- Welcome message: *"Welcome! Tell me what meal you're planning and I'll suggest 5 authentic South Indian recipes."*

#### `sous-chef.json`
- Tools: all (`*`), with `read` and `write` allowed
- Resources: `src/utils/recipe-template.ts`, `recipes/**/*.md`, `logs/transaction-log.md`
- Welcome message: *"Ready to prep! Point me to a recipe file and I'll generate a detailed prep list."*

### Steering Files

#### `executive-chef.md`
Persistent rules governing every Executive Chef interaction:
- Conversation parsing: meal type, dietary needs, available ingredients
- Authenticity enforcement: individual whole spices only, blocked ingredient table
- Caching logic: check `recipes/` first, tag cached vs new, generate dietary variants
- Top-5 suggestion format: numbered list with name, description, and cached/new tag
- Recipe output: 6-section markdown template written to `recipes/{kebab-case-name}.md`
- Sous Chef orchestration: mandatory after every recipe write
- Transaction logging: structured entries appended to `logs/transaction-log.md`

#### `sous-chef.md`
Persistent rules for the Sous Chef Agent:
- Input: recipe file from `recipes/` with all 6 required sections
- Output: `prep-list.md` with ingredient prep table and numbered steps
- Measurement standards: exact quantities (cups, tablespoons, teaspoons, grams) — no vague amounts
- One prep entry per ingredient, steps numbered sequentially with timing
- Transaction logging after each prep-list generation

#### `skills/substitution-skill.md`
On-demand skill invoked when a user says an ingredient is unavailable:
- 12 substitution mappings with conversion ratios and notes
- Case-insensitive matching, returns all alternatives for a given ingredient
- Blocked ingredients are never valid substitutes
- Unmapped ingredients return an empty list

## Transaction Log

The `logs/transaction-log.md` file records a timestamped audit trail. Each entry includes the agent name, request summary, files read/written, recipes suggested/selected, substitutions applied, and errors.

Example interactions logged:
- Lunch recipe suggestions using kothavaranga and onion → 5 new suggestions → Kothavaranga Thogayal selected and written
- Substitution request for urad dal and coconut → Chana dal (1:1) and Desiccated coconut (0.5x) applied
- Sous Chef prep-list generation for Kothavaranga Thogayal
- Idli and Tiffin Sambar recipe request → cached Sambar + 4 new suggestions → Idli and Tiffin Sambar written
- Tamarind unavailable for Tiffin Sambar → Amchur substitution (1:0.3) applied
- Sous Chef prep-list generation for Idli and Tiffin Sambar

## Prep List

The `prep-list.md` file at the workspace root contains the latest prep list generated by the Sous Chef Agent. It includes:
- An ingredient prep table with exact measurements and preparation actions (wash, soak, chop, measure, grind, etc.)
- Numbered steps with precise quantities and timing (e.g., "soak for 4–6 hours", "fry for 2 minutes", "steam for 10–12 minutes")

The current prep list covers Idli and Tiffin Sambar.
